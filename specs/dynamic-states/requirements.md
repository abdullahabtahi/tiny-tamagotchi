# Requirements вҖ” Dynamic States

## Behavioral Requirements

### R1 вҖ” Attention Pulse (Warning)
When any stat drops below **30** and the pet is in Normal state, the pet sprite enters a pulsing animation (CSS shake or glow). This is a visual-only warning. It does not change `state`. It persists as long as any stat remains below 30.

### R2 вҖ” Sick Transition
When any stat drops below **20**, the pet transitions to `state: "sick"`. This applies whether the stat reached 20 via decay or catch-up.
- `sickAt` is set to `Date.now()` at the moment of transition (if not already set).
- The sick sprite indicator is shown.

### R3 вҖ” Recovery Condition
The pet recovers from Sick to Normal when **all three stats** (Hunger, Happiness, Energy) are **вүҘ 50**, achievable through any combination of Feed, Play, and Rest actions.
- On recovery: `state` вҶ’ `"normal"`, `sickAt` вҶ’ `null`.

### R4 вҖ” Evolution Window
The pet can evolve from Normal state only. The evolution window:
1. Starts (`evolvedAt = Date.now()`) when all three stats are **> 80** simultaneously.
2. Resets (`evolvedAt = null`) if any stat drops **вүӨ 80** during the window.
3. Triggers evolution when `Date.now() - evolvedAt >= 18 * TICK_INTERVAL_MS` (18 consecutive ticks with all stats > 80).

### R5 вҖ” Evolution Trigger
When the evolution condition fires (R4 step 3):
- `state` вҶ’ `"evolved"`
- Stat decay is permanently paused.
- The evolved sprite and visual effect are shown.
- If the pet had ever been Sick before evolving (`sickAt` was non-null at some point), `recoveredFromSick` is set to `true`.

### R6 вҖ” Evolved State Permanence
Once `state === "evolved"`, no further state transitions occur. The pet remains evolved for the session. Decay is paused.

### R7 вҖ” Exclusive States
Exactly one state is active at all times: `"normal"`, `"sick"`, or `"evolved"`. There is no intermediate state.

### R8 вҖ” Visual Indicators
Each state has a distinct visual on screen:
- **Normal**: standard pet sprite (`(^В·ПүВ·^)`). State badge: `NORMAL`. Animation class determined by idle tier (see personality/requirements.md R4).
- **Sick**: sick sprite variant (`(Г—п№ҸГ—)`) with CSS class `.pet--sick` (`filter: grayscale(80%) sepia(20%)`). State badge: `SICK рҹӨ’`.
- **Pulse** (computed in render, not a state): CSS class `.pet--pulse` applied to sprite container when any stat < 30 and `state === "normal"`. Box-shadow pulses using `--color-danger` (`#e05c5c`). Removed when no stat is below 30.
- **Evolved**: alternate sprite (`(вҳ…Пүвҳ…)`) with CSS class `.pet--evolved` (`filter: drop-shadow(0 0 8px #e0a82e)` + 1.2s rotation loop). State badge: `EVOLVED вңЁ`.
- **Evolved + Resilient**: same as Evolved but badge reads `RESILIENT рҹҸ…` (when `recoveredFromSick === true`) and sprite face is `(вңҰПүвңҰ)`.

All CSS class keyframe contracts are defined in `specs/tech-stack.md` вҖ” CSS Animation Spec table.

### R9 вҖ” Reduced Motion
When `prefers-reduced-motion` is active, all CSS keyframe animations are disabled. State changes remain distinguishable through static class differences (different sprite text, different state badge text, greyscale filter for sick).

### R10 вҖ” Audio on State Transitions
Audio plays on exactly one state transition: **Normal вҶ’ Evolved**. On that transition, the synthesised **Evolve clip** plays once вҖ” a four-note fanfare (C5 вҶ’ E5 вҶ’ G5 вҶ’ C6) generated by the Web Audio API synthesiser in `src/audio.js`. Subject to the mute toggle.

Audio does **not** play on:
- Normal вҶ’ Sick (the greyscale visual is sufficient; adding a sound would startle).
- Sick вҶ’ Normal (recovery is gentle and continuous; no transition moment warrants a cue).
- The attention pulse appearing or disappearing.

This asymmetry is intentional and aligned with the calm-companion creative direction defined in `specs/tech-stack.md` вҖ” Audio Stack.

## Edge Cases

| Scenario | Expected Behavior |
|---|---|
| Two stats drop below 20 simultaneously | One `sick` transition fires; `sickAt` set once |
| Stat is exactly 20 | Not sick (threshold is **below** 20, i.e., < 20) |
| Stat is exactly 80 during evolution window | Evolution timer resets (threshold is **above** 80, i.e., > 80) |
| Pet sick when catch-up runs | Catch-up ticks evaluated tick-by-tick; sick state can trigger mid-catch-up |
| Evolution timer running, tab closed for 18 ticks | On reload, catch-up evaluates each tick; evolution may trigger during catch-up |
| Pet Sick, all stats at exactly 50 | Recovery triggers (вүҘ 50 condition met) |
| Pet Sick, Hunger = 50, Happiness = 40 | No recovery (not all stats вүҘ 50) |
