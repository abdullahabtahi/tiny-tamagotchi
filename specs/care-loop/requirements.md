# Requirements — Care Loop

## Behavioral Requirements

### R1 — Feed Action
Clicking the Feed button increases `hunger` by **20 points**, clamped to a maximum of **100**.

### R2 — Play Action
Clicking the Play button increases `happiness` by **15 points**, clamped to a maximum of **100**.

### R3 — Rest Action
Clicking the Rest button increases `energy` by **25 points**, clamped to a maximum of **100**.

### R4 — Single-Stat Rule
Each action affects **exactly one stat**. No action produces secondary stat changes.
- Feed does not change happiness or energy.
- Play does not change hunger or energy.
- Rest does not change hunger or happiness.

### R5 — Immediate Visual Feedback
Each button provides visible feedback within **200ms** of being clicked (e.g., a CSS scale or color transition). The stat bar updates immediately on the same render cycle.

### R6 — Capping Behavior
If a stat is already at or near 100, the action still executes and the stat is clamped:
- Hunger at 95 + Feed (+20) = 100 (not 115).
- Hunger at 100 + Feed (+20) = 100 (no change in displayed value).

### R7 — lastInteractedAt Update
Every action updates `lastInteractedAt` to `Date.now()` in the state object. This is required by the Personality feature's poop timer.

### R8 — State Persisted After Action
After any action, the updated state (including the new stat value and `lastInteractedAt`) is written to `localStorage` under key `tamagotchi_state`.

### R9 — Immutability
Action functions return new state objects. They do not mutate the state passed in.

### R10 — Audio Feedback
Each action plays a short synthesised audio cue at the same moment the stat updates, subject to the mute toggle and the audio stack rules in `specs/tech-stack.md` — Audio Stack.

Audio is generated by the **Web Audio API synthesiser** in `src/audio.js` — no asset files required. Clips are square-wave oscillator sequences:
- **Feed** → C5 → E5 (happy ascending blip, ~150ms total)
- **Play** → C5 → E5 → G5 (cheerful arpeggio, ~225ms total)
- **Rest** → G4 → E4 → C4 (descending lullaby, ~225ms total)

When `tamagotchi_muted === true`, the `playClip()` call is skipped entirely — the AudioContext is not touched. Master volume per oscillator is **0.18** (18% of browser volume). Each clip uses a 5ms linear attack and 10ms release envelope to prevent click artifacts.

## Edge Cases

| Scenario | Expected Behavior |
|---|---|
| Feed clicked when Hunger = 100 | Hunger stays 100; no error; feedback animation still plays |
| Feed clicked when Hunger = 95 | Hunger becomes 100, not 115 |
| Play clicked rapidly 5 times | Happiness increases by 15 per click up to 100, then stays at 100 |
| Rest clicked when Energy = 0 | Energy becomes 25 |
| Rest clicked when Energy = 80 | Energy becomes 100 (25 would be 105, clamped to 100) |
| Action taken while pet is Sick | Action still applies; may contribute to recovery if all stats reach ≥ 50 |
| Action taken while pet is Evolved | Action still executes; stat updates; decay remains paused |
